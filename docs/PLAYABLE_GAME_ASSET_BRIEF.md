# Playable Game Asset Brief

Creature Sandbox is currently a 2D canvas game with manifest-driven SVG sprite sheets. Keep gameplay-facing asset keys stable (`creature_*`, `food_*`, `prop_*`, `environment_*`, `particle_*`) and avoid using filenames as game logic.

## Imagegen Direction

Use case: stylized-concept
Asset type: 2D browser game scenario key art and sprite reference
Primary request: cohesive creature ecosystem game assets for playable scenarios
Scene/background: dark readable sandbox field with clear food, calm zones, props, and small expressive creatures
Subject: herbivores, predators, aquatic, flying, burrowing variants, food patches, calm zones, bounce/spring/fan/launch props
Style/medium: polished stylized game art, sprite-sheet friendly silhouettes, clean shapes, strong read at small sizes
Composition/framing: isolated assets and one top-down scene reference, no UI text baked into art
Lighting/mood: soft rim lighting, readable contrast, playful but restrained
Color palette: dark field, cyan/green ecosystem cues, purple scenario/god-mode accents, warm food highlights
Constraints: no watermark, no logos, no tiny text, no photorealism, keep silhouettes distinct at gameplay zoom

Live image generation requires `OPENAI_API_KEY`; if unavailable, continue with the existing SVG sprite generator and keep this brief as the source prompt.

## Generated Runtime Assets

- `ui_scenario_cards`: deterministic SVG-strip key art for the seven playable scenarios, generated by `creature-sim/scripts/generate-sprite-sheets.mjs` and consumed by the Scenario Director panel.
- Runtime path: `creature-sim/assets/sprites/ui/ui_scenario_cards.svg`.
- Manifest entry: `creature-sim/assets/sprites/sprite-manifest.json`.

## Future 3D Policy

If Creature Sandbox later adds 3D content, ship runtime assets as GLB or glTF 2.0 with stable names, applied transforms, gameplay pivots, collision proxies, and meshopt/KTX2-style optimization where supported. Do not mix raw DCC exports into the browser runtime.
